# 系统架构

## 三层产品结构

- `NearbyLove`：居民、物业、居委会共用的网页体验端，围绕附近行动组织表达、履约、协调与结果验证。
- `CommunitySense`：面向镇政府的治理大屏，以社区为粒度呈现人口、关系连接与行动趋势。
- 公共空间 AI 感知：可选硬件输入层，只产生匿名公共空间场景线索；不识别个人、不输出自动处罚结论。

## 演示模式

网页原型使用预置 Mock 数据，评委无需配置账号、密钥或硬件即可体验。Flask 后端和硬件脚本保留为可本地运行的技术验证代码：传感器读数进入区域汇总，人工确认后再形成事件或任务。

## 数据边界

- 虚拟户口本用于表达实际居住关系，不是法定户籍。
- AI 只辅助摘要、引导、生成待核实线索；不自动决定、不自动上门、不直接处罚。
- 公共空间感知不做人脸、身份、轨迹或个体行为识别。
- 正式信件、工单与通知反馈应在真实部署时由具备权限的主体处理并留痕。
